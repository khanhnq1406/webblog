# Designing Data-Intensive Applications

A structured 4-week study guide based on Martin Kleppmann's *Designing Data-Intensive Applications* — one of the most important books for engineers working with data at scale.

## What This Book Is About

Modern applications are rarely limited by CPU power. The real challenges are **data** — how much of it, how complex it is, and how fast it changes. This book explores the fundamental ideas behind building systems that are:

- **Reliable** — working correctly even when things go wrong
- **Scalable** — handling growth in data, traffic, and complexity
- **Maintainable** — easy for teams to work with over time

## Key Themes

Throughout the book, several themes recur:

**Data models matter.** The relational model, document model, graph model, and column-oriented storage each fit different problems — there is no universal winner.

**Storage engines have trade-offs.** Log-structured engines (LSM-trees) and page-oriented engines (B-trees) optimize for different workloads.

**Distributed systems are fundamentally unreliable.** Networks drop packets, clocks drift, and processes pause unpredictably. Good systems are designed around these realities, not in denial of them.

**There is no such thing as "the" consistency.** Linearizability, causal consistency, eventual consistency, and serializability are different guarantees with different costs.

**Batch and stream processing are converging.** The Lambda Architecture and its successors blur the line between historical data processing and real-time event handling.
